# プロジェクト概要: 宇宙船シューティングゲーム

このプロジェクトは、ブラウザおよびモバイルで動作する、高速で縦型の宇宙船シューティングゲームです。

## 📁 最適化されたディレクトリ構成案（ES6モジュール分割）
ファイルの役割ごとにディレクトリを細分化し、ES6の import/export を使ってモジュール化する構成です。（※Viteなどの軽量バンドラーの導入も視野に入れた構成です）

```text
.
├── index.html            # メインHTML（モジュールとして main.js を読み込む）
├── style.css             # 全体スタイリング（Tailwind設定なども含む）
├── manifest.json         # PWA設定
├── sw.js                 # サービスワーカー
│
├── assets/               # アセット類
│   ├── images/           # 画像（player, enemy, boss 等）
│   └── audio/            # （将来用）BGM・SEファイル
│
└── src/                  # 🌟 JavaScriptソースコード群
    ├── main.js           # エントリーポイント。ゲームの初期化と開始。
    │
    ├── core/             # ゲームの基盤となるシステム
    │   ├── Game.js       # ゲームループ制御（requestAnimationFrame）
    │   ├── Input.js      # キーボード・タッチ入力の監視と状態管理
    │   └── State.js      # スコア、EXP、レベル、プレイ状態の管理
    │
    ├── entities/         # 画面に登場するオブジェクト（クラス定義）
    │   ├── Player.js     # 自機クラス（移動、武器発射、被弾処理）
    │   ├── enemies/      # 敵クラス群
    │   │   ├── Enemy.js      # 敵の基底クラス / 通常敵
    │   │   ├── SmallEnemy.js # 高速な小型敵
    │   │   └── Boss.js       # ボス（霧演出、タイマー処理）
    │   └── Bullet.js     # 弾クラス（normal, spread, laser, homing）
    │
    ├── systems/          # ロジック・計算処理
    │   ├── Collision.js  # 当たり判定（AABB計算処理）
    │   ├── Spawner.js    # 敵の出現制御（レベルごとの出現テーブルなど）
    │   └── Renderer.js   # Canvas描画処理（動的リサイズ対応含む）
    │
    ├── utils/            # 汎用的な便利関数
    │   ├── math.js       # 距離計算、角度計算など
    │   └── constants.js  # 定数（画面比率設定、Zインデックス、ステータス初期値など）
    │
    └── ui/               # DOM操作・UI制御
        ├── HUD.js        # スコア・HPバーの更新
        └── Menus.js      # レベルアップ画面、強化選択UI、ゲームオーバー画面
```

### 💡 この構成にする3つのメリット
1. **「何がどこにあるか」が直感的にわかる（認知負荷の低下）**
「当たり判定のバグを直したい」ときは `systems/Collision.js` を見ればよく、「新しい武器を追加したい」ときは `entities/Player.js` と `entities/Bullet.js` を触るだけで済みます。1400行のスクリプトをスクロールして探す手間がなくなります。

2. **定数・設定の外部化によるバランス調整の容易化**
`utils/constants.js` に「敵のHP」「自機の初期スピード」「武器の威力」などのマジックナンバー（直接書かれた数値）を集約することで、ゲームバランスの調整テストが圧倒的に楽になります。

3. **将来の拡張が圧倒的に楽になる**
- 新ボスの追加: `Boss.js` を継承して `BossStage2.js` を作るだけで実装できます。
- サウンドの実装: 今後BGMやSE（効果音）を追加する際も、`core/AudioManager.js` を新規作成して組み込むだけで、既存のコードを汚さずに拡張できます。

## 🛠 移行ステップ（リファクタリングの進め方）
いきなりすべてのファイルを分割すると動かなくなるリスクが高いです。以下の順序で徐々に移行することをお勧めします。

1. **モジュール化の準備**:
   `index.html` の script タグを `<script type="module" src="src/main.js"></script>` に変更します。（ローカルで動かすにはVSCodeの「Live Server」などのローカルサーバー環境が必要です）
2. **定数・ユーティリティの切り出し**:
   まずは画面サイズ設定や AABB判定の関数など、独立した処理を `utils/` に移動させます。
3. **クラスの切り出し**:
   Player, Enemy, Boss, Bullet クラスをそれぞれ1ファイルに切り出し、`export class Player...` のように書き出します。
4. **コアシステムの切り出し**:
   最後にゲームループ全体や入力処理を切り離します。

## 🎮 ゲーム仕様

### 基本システム
- **レンダリング**: HTML5 Canvasを使用した毎秒60フレームの描画。
- **操作方法**:
    - **デスクトップ**: WASDキー または 矢印キーで移動。
    - **モバイル**: 画面タッチ・スワイプによる直感的な移動。ダブルタップでフルスクリーン切り替え。
- **進行**: 敵を倒してスコアと経験値（EXP）を獲得。レベルアップにより自機が強化される。

### 強化システム（レベルアップ）
レベルアップ時に以下のいずれかの強化がランダムまたは段階的に適用される：
- **弾数増加**: 同時に発射される弾の数が増える。
- **武器タイプ変更**: 
    - `normal`: 標準的な弾。
    - `spread`: 広範囲に拡散する弾。
    - `laser`: 貫通力のある太いレーザー。
    - `homing`: 敵を追尾する弾。
- **連射速度向上**: 弾の発射間隔が短縮される。

### 敵キャラクター
- **通常敵 (Enemy)**: 上部から出現し、時折弾を撃ちながら下下してくる。
- **小型敵 (Small Enemy)**: 高速で移動し、より頻繁に攻撃してくる。
- **ボス (Boss)**: レベル10ごとに登場。高いHP、多彩な攻撃パターン、霧（Fog）演出、そして「30秒以内に撃破しないと敗北」という制限時間（Death Timer）を持つ。

### PWA機能
- `manifest.json` と `sw.js` により、スマートフォンにアプリとしてインストール（ホーム画面に追加）が可能。
- オフライン環境でも動作するように設計。
